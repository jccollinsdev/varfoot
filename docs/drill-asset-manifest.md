# Drill asset manifest

Every drill in [`src/data/drillCatalog.ts`](../src/data/drillCatalog.ts) ships with a
generated SVG diagram at `imagePath` (`/drills/{id}.svg`, served from
[`public/drills/`](../public/drills/)). Verified 2026-06-07:

- scored drills in the catalog: `50`
- drill SVG files present in `public/drills/`: `50`
- missing asset paths: `0`

All 50 files are produced — and can be regenerated byte-for-byte — by:

```bash
node scripts/generate-drill-diagrams.mjs
```

The script clears `public/drills/*.svg` and re-renders all 50 from a small set of
reusable **scene templates** (see below), so regenerating never leaves stale files
behind and every `imagePath` in the catalog is guaranteed to resolve to a real file
(`diff` of catalog ids vs. generated filenames is empty — verified 2026-06-07).

## Visual style contract

- **Canvas:** 360×240 rounded dark card (`#101216` fill, `#262a30` border) — drops
  cleanly onto the app's dark theme without its own background showing through.
- **Neon green (`#39ff73`)** — the primary action: the pass, the touch, the sprint
  line, the target zone, the timer ring. If it's the thing the player is being
  graded on, it's green.
- **Muted gray/white (`#9aa1a9` / `#c7cbd1`)** — the player ("You"), static
  equipment (walls, goals, start cones), and reference lines.
- **Amber accent (`#ff9d4d`)** — secondary cues only: cones that mark a course,
  defenders/partners, arrows that show setup rather than the graded action.
- Every diagram carries a short uppercase caption naming the core action (e.g.
  "20 PASSES — CUSHION DEAD", "HOLD UNTIL FORM BREAKS"), and an
  `aria-label` of `"{drill name} diagram"` for accessibility.

## Scene templates

Fifty distinct diagrams are produced from 14 parameterized templates so the set
stays visually consistent while every drill still gets a layout that matches what
it actually measures:

| Template | What it shows | Used for |
|---|---|---|
| `wallRebound` | Player passes into a wall, ball rebounds back | Wall-based first-touch / combo / weak-foot drills |
| `throwAndControl` | Ball thrown up, dropped, cushioned on the ground | High Ball Drop-Dead |
| `gatePass` | Two cones forming a gate, ball driven through from a marked distance | Gate Pass, Driven Ball Switch |
| `targetSquare` | Dashed target zone with a looping ball arcing into it | Long Ping (30-step) |
| `coneWeave` | A line of cones with a weaving dribble path (slalom / diamond / figure-8 / square) | Cone Slalom, Inside-Outside Cut, Figure-8, One-Touch Square |
| `shieldBox` | Marked box with a shielding player and a pressing partner/defender | Box Shield, 1v1 Close-Down |
| `sprintLane` | Two cones, a directional sprint line, optional turn arc / partner / ball | Recovery Sprint, Give-and-Go, Pro Agility Shuttle, 30-Yard Sprint, Recovery Run Reaction, Blind-Side Run, Overlap & Return |
| `defendingLane` | Attacker with ball approaching a defender in a marked lane | Jockey, Poke-Tackle, Sliding Tackle |
| `aerial` | Crossed/tossed ball met in the air near a goal or fence | Header Clearance, Crossing & Far-Post Run, Flick-On Header Control |
| `hold` | Static hold pose (plank / wall sit / single-leg balance) with a capped progress ring | Plank Hold, Wall Sit Hold, Single-Leg Balance |
| `finishing` | Goal with dashed corner target zones and a shot trajectory | Power Strike, First-Time Finish, Curl-and-Place Free Kick, Weak Foot Finishing |
| `ballMastery` | Player with looping touch arcs above the ball | Juggling, Sole Roll, Weak-Foot Juggling, Field Scan, Reaction Ball Drop |
| `shuttleTest` | Marked 20 m lane with turn markers, runner ahead of a "beep" | Beep Test, Yo-Yo Intermittent |
| `bodyweight` | Bodyweight rep figure (push-up / jump / rotation) with rep cue | Max Pushups, Plyo Box Jumps, Core Plank Rotation, Standing Long Jump |
| `checkin` | Iconographic daily-habit tile (fuel / water / sleep) | Daily Calories, Daily Water, Sleep Duration, Hydration Check-In, Sleep & Recovery Routine |

## Regeneration / extension notes

- To add a new drill, add its entry to `drillCatalog.ts` **and** a matching entry
  in the `drillScenes` map in `scripts/generate-drill-diagrams.mjs` (id → `[templateName, params]`),
  then re-run the generator. The script throws if a catalog id has no scene mapping,
  so a missing diagram fails the build loudly rather than silently 404ing in the app.
- `params.note` becomes the diagram's caption; keep it short (the renderer
  truncates captions over 46 characters) and written as an imperative cue
  ("20 PASSES — CUSHION DEAD") rather than a restatement of the drill name.
